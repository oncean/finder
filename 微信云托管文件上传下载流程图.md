# 微信云托管文件上传下载流程图

> 基于服务端两阶段上传与小程序直传的完整链路

---

## 一、服务端上传流程（两阶段）

```mermaid
flowchart TD
    subgraph Client["客户端"]
        A["选择文件<br/>wx.chooseImage / form"]
    end

    subgraph Backend["后端服务 NestJS 云托管容器"]
        B["接收文件<br/>POST /api/v1/storage/upload"]
        C{"CBR_ENV_ID<br/>是否存在?"}
        D["获取上传链接<br/>POST http://api.weixin.qq.com/tcb/uploadfile"]
        E{"返回结果<br/>errcode = 0?"}
        F["构造 COS 表单<br/>FormData + Signature + Token"]
        G["直传文件到 COS<br/>POST cos.ap-shanghai.myqcloud.com"]
        H["返回 fileId + url"]
        I["返回 500 错误<br/>CBR_ENV_ID 未设置"]
        J["返回 400 错误<br/>获取链接失败"]
        K["返回 500 错误<br/>COS 上传失败"]
    end

    subgraph COS["腾讯云 COS"]
        L["对象存储<br/>cloud://envid.xxx/path"]
    end

    subgraph DB["数据库"]
        M["存储 fileId / url<br/>关联业务记录"]
    end

    A --> B
    B --> C
    C -- 否 --> I
    C -- 是 --> D
    D --> E
    E -- 否 --> J
    E -- 是 --> F
    F --> G
    G --> L
    G -- 失败 --> K
    L --> H
    H --> M
    H --> Client
```

**关键步骤说明**

1. **阶段一：获取上传链接** — 后端调用微信内网接口 `http://api.weixin.qq.com/tcb/uploadfile`，传入环境 ID 和目标路径，返回 COS 临时上传地址、签名和 Token
2. **阶段二：直传 COS** — 后端构造 multipart/form-data 请求，将文件直接上传至腾讯云 COS，不经过后端中转文件流
3. **免鉴权** — 云托管容器内调用微信接口无需 access_token，自动识别环境身份

## 二、小程序直传流程

```mermaid
flowchart TD
    subgraph MiniApp["微信小程序"]
        A1["选择本地文件<br/>wx.chooseImage"]
        B1["调用 SDK 上传<br/>wx.cloud.uploadFile"]
        C1{"基础库 >= 2.13.2?"}
        D1["上传成功<br/>返回 fileID"]
        E1["上传失败<br/>返回错误码"]
        F1["提示升级基础库"]
    end

    subgraph Cloud["微信云托管"]
        G1["验证环境 ID"]
        H1["分配 COS 上传链接"]
        I1["直传文件"]
        J1["写入对象存储"]
    end

    subgraph Storage["对象存储"]
        K1["cloud://envid.xxx/path"]
    end

    A1 --> B1
    B1 --> C1
    C1 -- 否 --> F1
    C1 -- 是 --> G1
    G1 --> H1
    H1 --> I1
    I1 --> J1
    J1 --> K1
    K1 --> D1
    I1 -- 网络异常 --> E1
```

**与后端上传的区别**

- 小程序端 SDK 自动完成「获取链接 + COS 上传」两阶段，开发者无需关心内部实现
- 适合用户直接上传场景（如头像、图片发布），减少后端带宽消耗
- 大文件或需要业务校验的场景，建议先传后端再转存

## 三、文件下载/访问流程

```mermaid
flowchart TD
    subgraph Request["请求方"]
        A2["需要访问文件<br/>如图片展示"]
        B2["使用 fileID 或 HTTPS URL"]
    end

    subgraph Way1["方式一：HTTPS 直链"]
        C2["拼接固定 URL<br/>https://envid.tcb.qcloud.la/path"]
        D2["CDN 加速访问"]
    end

    subgraph Way2["方式二：临时下载链接"]
        E2["调用接口获取<br/>tcb/getUploadTcbFileLink"]
        F2["返回带签名的<br/>临时下载 URL"]
        G2["限时访问<br/>默认 15 分钟"]
    end

    subgraph Way3["方式三：小程序 SDK"]
        H2["wx.cloud.downloadFile"]
        I2["传入 fileID"]
        J2["下载到本地临时文件"]
    end

    A2 --> B2
    B2 --> C2
    C2 --> D2
    B2 --> E2
    E2 --> F2
    F2 --> G2
    B2 --> H2
    H2 --> I2
    I2 --> J2
```

**三种访问方式对比**

| 方式 | 适用场景 | 有效期 | 权限控制 |
|------|----------|--------|----------|
| HTTPS 固定链接 | 公开资源（头像、封面图） | 永久 | 公开访问 |
| 临时下载链接 | 私有文件（合同、身份证） | 15 分钟 | 签名鉴权 |
| 小程序 SDK | 小程序内部下载 | 一次性 | 微信生态鉴权 |

## 四、完整业务时序图

```mermaid
sequenceDiagram
    autonumber
    participant C as 小程序/前端
    participant B as NestJS 后端
    participant WX as 微信云托管<br/>内网接口
    participant COS as 腾讯云 COS
    participant DB as PostgreSQL

    C->>B: POST /api/v1/storage/upload<br/>multipart file
    B->>B: 校验文件类型、大小
    B->>WX: POST /tcb/uploadfile<br/>{env, path}
    WX-->>B: {url, token, authorization, file_id}
    B->>COS: POST url<br/>FormData(file, Signature, Token)
    COS-->>B: 200 OK
    B->>DB: INSERT uploads(fileId, url, cloudPath)
    B-->>C: {fileId, url, cloudPath}

    Note over C,COS: 后续访问文件
    C->>C: 使用 url 或 fileId<br/>展示/下载文件
    alt 固定 HTTPS 链接
        C->>COS: GET https://envid.tcb.qcloud.la/path
        COS-->>C: 文件内容
    else 小程序 SDK 下载
        C->>WX: wx.cloud.downloadFile({fileID})
        WX->>COS: 获取临时链接
        COS-->>C: 文件内容
    end
```

## 五、错误处理流程

```mermaid
flowchart TD
    A["开始上传"] --> B{"检查环境"}
    B -->|CBR_ENV_ID 缺失| C["记录警告日志<br/>返回 500"]
    B -->|环境正常| D{"获取上传链接"}
    D -->|errcode != 0| E["记录错误日志<br/>返回 400"]
    D -->|链接获取成功| F{"COS 直传"}
    F -->|网络超时| G["重试 3 次"]
    G -->|仍失败| H["记录错误日志<br/>返回 500"]
    G -->|重试成功| I["上传完成"]
    F -->|直传成功| I
    I --> J["写入数据库<br/>返回结果"]
```
